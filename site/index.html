> what is this

A Python pipeline for extracting, storing, and querying factual claims from long-form research sources — books, articles, depositions, transcripts, court documents. No database server, no external dependencies, no framework lock-in. Just files and stdlib.

Facts live in facts.jsonl, a flat JSONL file where every line is a structured claim with a date, certainty score, source citations, and entity tags. Once populated, the JSONL migrates to SQLite for richer querying — timelines, entity graphs, theme threads.

The extraction engine scores sentences from source documents against configurable indicators of factual density: dates, named entities, monetary figures, verifiable events. Low-scoring sentences are discarded. High-scoring ones become facts.

Built for investigative journalism and documentary research. Tested on a multi-year, 300+ source investigation. The schema is generic — bring your own sources, configure your own entities, define your own themes.

The pipeline has two modes. During active research, facts live in JSONL — fast to append, easy to diff, human-readable. When the corpus stabilizes, migrate_to_sqlite.py loads the JSONL into a relational SQLite database with full entity and citation linking.

> getting started

Requirements: Python 3.11+ (uses stdlib tomllib). No pip installs needed.

# 1. Clone the repo
git clone https://github.com/luxwon/investigative-pipeline
cd investigative-pipeline

# 2. Configure your project paths
cp config/project.toml.example .env
# Edit .env:
#   PIPELINE_VAULT=/path/to/your/project
#   PIPELINE_ARCHIVE=/path/to/your/data
source .env

# 3. Add your source configs
cp config/sources/example-source.toml config/sources/my-source.toml
# Edit my-source.toml with your book/article metadata

# 4. Put a book in books/ (pandoc from EPUB recommended)
pandoc ~/Downloads/my-source.epub -t markdown -o books/my-source.md

# 5. Extract facts
python3 src/extract_facts.py books/my-source.md

# 6. Build source registry
python3 src/build_registry.py

# 7. Validate
python3 src/validate.py

# 8. Generate views
python3 src/generate_views.py

# 9. Migrate to SQLite (after corpus is stable)
python3 src/migrate_to_sqlite.py

# 10. Search
python3 src/search_facts.py "keyword query"
python3 src/search_facts.py --person "Jane Smith" --year 2003
python3 src/search_facts.py --certainty 8 --full

Entity aliases go in config/entities.toml (copy from entities.toml.example). Source metadata goes in config/sources/*.toml, one file per source. The pipeline picks them up automatically on next run.

> data formats

All fact data is stored as newline-delimited JSON (.jsonl). One object per line. Every line is a complete, independent record.

facts.jsonl — one fact per line:

{
  "id":            "F-2005-0042",
  "claim":         "John Doe transferred $3.2M to the Cayman entity on March 14, 2005.",
  "date":          { "value": "2005-03-14", "precision": "exact" },
  "certainty":     8,
  "status":        "confirmed",
  "review_status": "reviewed",
  "sources": [
    { "source_id": "SRC-BOOK-smith-dark-money", "page": "142" }
  ],
  "people":        ["John Doe"],
  "places":        ["Cayman Islands"],
  "organizations": ["Cayman Entity LLC"],
  "tags":          ["financial", "transfer"],
  "created":       "2025-01-15",
  "modified":      "2025-01-15"
}

Field reference:

id F-{YEAR}-{NNNN}. Year is the event date; NNNN is a zero-padded sequence.
claim A single, complete factual statement. Should be independently verifiable.
date.value ISO date string: YYYY-MM-DD, YYYY-MM, or YYYY.
date.precision exact | month | year | range | undated | approximate
certainty 0–10. 8+ = court-documented. 5–7 = credible journalism. 1–4 = unverified/alleged.
status alleged | confirmed | disputed | disproven
sources Array of {source_id, page, quote}. source_id matches sources.jsonl.
people / places / orgs Entity arrays. Populated by alias resolution from config/entities.toml.
tags Free-form tags for topic clustering and theme association.

# Reads PIPELINE_VAULT and PIPELINE_ARCHIVE from environment
# or from a .env file in the current working directory.

import os
from pathlib import Path

def _load_dotenv(path):
    if not path.exists(): return
    for line in path.read_text().splitlines():
        line = line.strip()
        if not line or line.startswith("#") or "=" not in line: continue
        key, _, value = line.partition("=")
        key = key.strip(); value = value.strip().strip('"').strip("'")
        if key and key not in os.environ: os.environ[key] = value

_load_dotenv(Path.cwd() / ".env")

VAULT   = Path(os.environ.get("PIPELINE_VAULT",   ".")).expanduser().resolve()
ARCHIVE = Path(os.environ.get("PIPELINE_ARCHIVE", str(VAULT / "data"))).expanduser().resolve()

src/extract_facts.py — sentence scoring (key logic):

# Each sentence is scored against factual density indicators.
# Sentences above score_threshold are extracted as fact candidates.

FACTUAL_INDICATORS = {
    # Temporal markers
    r'\b(19|20)\d{2}\b':                         3,
    r'\b(January|February|March|…|December)\b':    2,
    # Numerical specificity
    r'\$[\d,]+(?:\.\d+)?(?:\s?(?:million|billion))?': 3,
    r'\b\d{1,3}(?:,\d{3})+\b':                    2,
    # Action verbs
    r'\b(signed|filed|arrested|hired|paid|founded)\b': 2,
    # Attribution
    r'\b(according to|testified|told investigators)\b': 1,
}

def score_sentence(sentence: str, source_bonus: int = 0) -> int:
    score = source_bonus
    for pattern, weight in FACTUAL_INDICATORS.items():
        if re.search(pattern, sentence, re.IGNORECASE):
            score += weight
    return score

thread          -- interpretive themes / narrative threads

packet_evidence -- packet → evidence links
